System Overview

The CodeB Internal Management System (IMS) provides secure Role-Based Access Control (RBAC) via JSON Web Tokens (JWT). The system currently features two main user roles for the authentication module:

  • Admin: Full system control over users and modules.
  • Sales Person: Restricted features and invoicing dashboard.

Requirements Met

  • Backend: Spring Boot 3.x REST API, MySQL Database (Tables implemented: User, Client, Group, Chain, Brand, Subzone, Estimate, Invoice, Payment).
  • Frontend: React 18 UI built with Vite and Bootstrap 5.
  • Security: Stateless JWT Authentication via Spring Security.

How to Use the Application

1. Registration

  • Navigate to the /register web page.
  • Fill out your Full Name, Email, Password, and select a Role (Admin / Sales Person).
  • Submit the form. Upon success, you will be redirected to the Login page.

2. Login

  • Navigate to /login.
  • Enter your registered Email and Password.
  • Upon successful authentication, the server generates a secure JWT token, and you are directed to the secure Dashboard.

3. Dashboard Navigation

  • The Dashboard reads your Profile information and greets you dynamically.
  • It displays varying UI blocks depending on whether you logged in as an Admin or a Sales Person.
  • Clicking Logout instantly revokes your local session and forces you back to the login screen, satisfying secure session management requirements.
